Hosting springboot+slq+docker app on ec2 isntance
====================================================
BACKEND WITH DATABASE
-----------------------
step1=> add docker file into your springboot app and add configurations into it
		> Dockerfile
		
step2=> build the spring boot app in local 
		> mvn clean package
		
step3=> build docker image locally 
		> docker build -t cms-backend .
			OR
		> docker-compose up --build -d [***if you use this command then below 2commands for runing docker and mysql will not be required to be executed]
			> then in this case you can not directly push this bundle to ec2, so Build and push only the Spring Boot image to Docker Hub and then create docker-compose file in ec2(refer the docker-compose.yml present below).
			> then docker pull kumarsurajp22/cms-backend > docker-compose up -d
		
step4=> run docker image locally inside docker container
		> docker run -d -p 8001:8001 cms-backend
		
step5=> Push Docker Image to Docker Hub
		=> Login to Docker Hub
			> docker login
			> give username, passowrd to login
		=> Tag the image
			> docker tag cms-backend kumarsurajp22/cms-backend
		=> Push it
			> docker push kumarsurajp22/cms-backend
			
step6=> conenct to ec2 amzon linux instance by mobaxterm

Note: for running commands on ec2 instance in a linux environment we require to use "sudo" before every command.

step7=> install docker
			> sudo yum update -y
			sudo yum install docker -y
			sudo service docker start
			sudo usermod -aG docker ec2-user
			
Note: java or sql are not required to be install inside ec2 instance as docker provides them.

step8=> login to docker using username and password

step9=> Pull Image on EC2 & Run
		> sudo docker pull kumarsurajp22/cms-backend
		> sudo docker run -d -p 8001:8001 kumarsurajp22/cms-backend
		
step10=> to create container inside docker
		> sudo docker run --name mysql-container -e MYSQL_ROOT_PASSWORD=root   -e MYSQL_DATABASE=collegemanagementapp   -p 3306:3306 -d mysql:8
		
step11=> used for running the mysql continer running on which port number
		> sudo docker inspect -f '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' mysql-container
		
		used for checking the mysql continer running on which port number
		> sudo docker inspect -f "{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}" mysql-container


step12=> to get inside musql shell for executing database oprations
		> sudo docker exec -it mysql-container mysql -u root -p

FRONTEND
----------
****Note: do all the below steps inside root folder
Step1: install cross env
		> npm install --save-dev cross-env

Step2: add cross-env before url, inside package.json file
		> example: "build:prod": "cross-env REACT_APP_API_URL=http://3.93.164.81:8001 react-scripts build" 

Step3: create build folder in to react app
		> npm run build:prod

Step4: tag the build,
		> docker build -t cms-frontend .
Step5: run the image inside container
		> docker run -d -p 8002:8002 cms-frontend
		***Note: default port number is 80 for frontend, to run the app in custom port you have to add nginx.conf file inside root

Step6: login into docker
		> docker login

Step7: tag the image to repository
		> docker tag cms-frontend kumarsurajp22/cms-frontend

Step8: push the image to repository
		> docker push kumarsurajp22/cms-frontend

Step9: install docker into ec2 if not installed
		> sudo yum update -y
			sudo yum install docker -y
			sudo service docker start
			sudo usermod -aG docker ec2-user
Step10: pull image from dockerhub
		> sudo docker pull kumarsurajp22/cms-frontend
Step11: run the container
		> sudo docker run -d -p 8002:8002 kumarsurajp22/cms-frontend

*) compose docekr file in ec2
	> nano docker-compose.yml
	>paste the docker compose file contents from local into ec2
	>Ctrl+O → Enter → Ctrl+X

install docker compose into ec2
	> sudo curl -L "https://github.com/docker/compose/releases/download/v2.24.5/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
> then give it, execute permissions
	> sudo chmod +x /usr/local/bin/docker-compose
> then hit the below command to pull the images from docker hub
	> sudo docker compose up -d
	[the above command docekr-compose up -d, 
	It starts all services defined in your docker-compose.yml,
	1. Reads docker-compose.yml,
	2. Pulls images from Docker Hub
	3. Creates containers from the specified images
	4. Starts containers
	5. Runs in detached mode (-d)]
	Note: if the above command is not working then hit 
	> sudo usermod -aG docker ec2-user
	then logout of docekr by
	> exit
	then re-login
	> docker login
	then hit again
	> docker compose up -d
------------------------------------
docker-compose.yml file ec2
----------------------------
services:
  mysql:
    image: mysql:8
    container_name: mysql-container
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: collegemanagementapp
    ports:
      - "3306:3306"
    healthcheck:
      test: [ "CMD", "mysqladmin", "ping", "-h", "localhost" ]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - backend-network

  backend:
    image: cms-backend
    container_name: cms-backend-container
    ports:
      - "8001:8001"
    environment:
      SPRING_PROFILES_ACTIVE: docker
    depends_on:
      mysql:
        condition: service_healthy
    networks:
      - backend-network

networks:
  backend-network:
    driver: bridge

#Bridge driver = default for Linux containers
#it will create a network named backend-network
#All containers (services) that use this network can talk to each other by name
#Containers not in the network can’t access them
